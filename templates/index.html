Web Automation at Work

Selenium, Tkinter

Coding is for automating the boring stuff, isn’t it? When I was learning web automation, I knew this is a great tool to reduce workload in my administrator related task. And oh boy, it did!

There is a database where I am the user admin. The tasks run in a browser and are quite manual and repetitive. So I have created a utility app which automates login, search, profile analysis, password generation and reset, renews account validity, creates emails for the specific cases and generates the file name for the archive.

These were some of my smaller challenges:

  • how to read the information from different fields
  • how to create formatted Outlook emails
  • designing the whole process
  • designing the GUI
  • debugging

Greater challenges:

  • how to make searches and handle the results
  • installing any packages on the company laptop
  • learning how to create an executable app from Python script

How to make searches and handle the results
Finding elements in this system wasn’t easy. ID, class name ore name didn’t work most of the time. I had to use the full XPath. I had to learn how to read the value of a checkbox: by is_selected(). When there was no way around to press a button, I made Selenium to press TAB a couple of times to reach the correct field.
The search result was also fuzzy. After filtering the list, I had to make different scenarios how to react when there are no results, there is 1 result and how to communicate with the user if there are many results.

Installing any packages on the company laptop
Installing anything was hardly possible as the network connection was very limited. I downloaded the packages and tried to install them “offline”, but they often need other packages from pypi to work. Figuring out what are the other packages and what is the necessary order of installation took time, even if the key user was helping me.
After several trials, the solution was that we decided to move everything to a virtual machine. There, I was able to install packages easily, however on my regular Windows I could not run my scripts without them. This leads us to the next point:

Learning how to create an executable app from Python script
As I couldn’t install the packages on my regular machine, I had to receive the app in executable form. Therefore, I used auto-py-to-exe for the conversion. I learned using it from 0.
First, Selenium was not included in the exe. It took me some time on Stackoverflow to find the code that I had to include in my script to make it work.
Second, now pyperclip was missing. I had to add it to the hidden imports. Finally, I was able to use my code.

The first real life coding project turned out to be a blast and speeds up my work by 70%. I love to use it day by day.

City Air Quality

Flask, WTForms, Requests (API)

My goal was to create a web app, where you search for a city, and the app gives you the AQI plus a funny GIF to cheer you up. We learn and understand faster from pictures than from numbers. If it makes you laugh, it is even better!

I chose the API service of aqicn.com and I got lost a bit in the different maps and tiles they offered. Most did not work for me. I decided to keep one map and one tile. The more challenging part was yet to come: how to send a request and how to take the useful data from the answer?

Image Colour Palette Generator

Flask, Numpy, PIL, Pandas

The goal of this project: analysing a picture pixel by pixel and finding the 10 most common values. I learned PIL, Numpy and Pandas, but how to start?

I created a minimal web app where file upload was possible. I used PIL to open the image. Numpy was my next tool, but I had to read the docs. So I created an array and got the shape of the image. Then I looped through all the pixels and saved them in a list.

Okay, so how to find the 10 most common values? Reading the docs again, but now the Pandas. The value counts function helped me out. The brain of the app was done.

I designed a simple site, where you can upload your image. The app displays it together with the 10 most common colours and their RGB and HEX codes. Project complete.

Data Science: Predict Earnings

Plotly, Seaborn, Matplotlib, Sklearn, Pandas, Numpy

Data analysis and projection on schooling, work experience and earnings.

The National Longitudinal Survey of Youth 1997-2011 dataset. My goal was to to find relationships between different skills and future earnings. Yet, I didn’t know what will I find in the set and how to understand such acronyms as ASWABWK. First, I spent some time reading about the background and the meanings of the variables.

After cleaning the data, I started to get to know the content. I created histograms of different skills, such as math, vocabulary, hourly wage, etc. The distributions were quite normal and I did’t find anything to grasp on. Then I started comparing the skills and earnings with pairplots and jointplots. I was curious to find any correlation. I found a weak correlation between verbal skills and earnings. But, I found researching schooling, work experience and earnings more promising.

There was a step up in earnings after 12 school years. Those, who study at least 12 years, can make more than those, who leave education early. After 16-17 years people are likely to earn the most.

The relations between experience and wage showed me a different result. What I found was, that after 3-4 years of experience people starting to earn more. But only after about 11 years the lower end of the wage range increases. This means, that after 11 years of experience, even those, who earn the least, get a raise.

I was curious how to make predictions based on the data. Using multivariable regression, I predicted the future earning of someone with 16 school years and 5 years of experience. The data had to be split between 80% training set and 20% test set. This is very interesting: for every extra year in school the hourly wage increased by $1.8.

My prediction shows, that after the mentioned education and experience the hourly wage is $19.7 in 2011.

The results are interesting, but as usual, the more we know, the more questions we ask. It is worth to continue the analysis between education and earnings.
